  • CEOT-DRL: a deep reinforcement learning (DRL) package
    Deep Learning, Deep Reinforcement Learning, Python, Pytorch, matplotlib

    CEOT-DRL is a deep reinforcement learning (DRL) package developed during my stay in the Centro de Electrónica, Optoelectronica e Telecomunicações (CEOT) research center: https://www.ceot.ualg.pt/ for DRL research on networking and systems. CEOT-DRL is designed such that it abstracts almost everything. Therefore, regardless of which computational graph platform is used to define the neural nets, it can still run smoothly.

    Inspired by the Matlab Reinforcement Learning Toolbox, CEOT-DRL decomposes the learning task into scenario, environment, model, and a training manager. This design allows us to focus on the problem to develop the required scenario and let the training manager does the whole end to end training and testing. The following example shows how to solve the CartePole-v1 problem using an Actor-Critic model trained by the training manager. The source code is available here: source code (this is the first experimental version and the latest version will be publicly available soon). 
    
        import sys
        import timeit
        
        from MLS.ceot_drl.scenarios import scenario_ac as s
        from MLS.ceot_drl.core.TrainingManager import TrainingManager as TM
        
        sys.path.append("..")

        def main():
            # define a training manager object
            tm = TM(s.num_episodes, 
                    s.episode_length, 
                    s.agent,
                    s.env,
                    log_file=s.log_file)
        
            print('Scenario:%s' % s.title)
            start = timeit.default_timer()
            # let it do the magic
            tm.run(verbose=False)
            end = timeit.default_timer()
            print('\n It took ~{} useconds'.format(str(round(end-start))))
        

        if __name__ == "__main__":
           main()
  • Sheba: a lightweight image processing library
    Java, NetBeans Platform, Efficient Java Matrix Library (EJML)

    A free open source lightweight image (and signal) processing package which is designed ground-up based on the best practices found in some existed packages like imagej and cellprofiler. It is designed to act as a container of components like channel, plugin, filter, and pipeline. It basically creates pipelines and executes them, which is helpful for fast prototyping, or developing a lightweight core module of an application that requires image processing. The main features of Sheba is its high modulation, and simplicity to create image processing pipelines. In Sheba the input and output are channels. An image, for instance, can contain multiple channels and can be processed by filters. To realize image processing pipelines, Sheba arranges them in a sequence of filters (i.e. a pipeline). For instance, if the task is to segment an image. This can be done by creating a pipeline that starts by a Gaussian filter (smoothing plugin), followed by a grayscale filter (conversion plugin), followed by a segmentation filter (segmentation plugin).

    The following example shows how to calculate a polynomial illumination field from a grayscale image. Sheba's full source code can be found here: source code 
    
      // build a pipeline to estimate the illumination surface of an image using a 2-D polynomial of polynomialDegree degree 
      Pipeline pipeline = new Pipeline();
      pipeline.addTask(new Scale(downSamplingFactor)); // scale down
      pipeline.addTask(new IlluminationSurface(IlluminationSurface.SurfaceType.Polynomial, polynomialDegree)); // estimate the illumination field
      pipeline.addTask(new Scale((1 / downSamplingFactor))); // scale up back    
      outputImage = imageFilter.run(inputImage, pipeline); // apply the pipeline on the image.
    Moreover, Sheba allows adding compiled plugins (.jar files) without changing the code or modifying the pipelines. That is adding or updating functionalities on the fly. This can be helpful when a team works on the pipeline where plugins can be tested locally and deployed to the final system.
  • P-TRAP: a panicle trait phenotyping tool
    Java, NetBeans Platform, OpenCV
    article| See the user manual

    A free open source application for high-throughput measurements of panicle architecture and seed-related traits. The software is written in Java and can be used with different platforms (the user-friendly Graphical User Interface (GUI) uses Netbeans Platform 7.3). The application offers three main tools: a tool for the analysis of panicle structure, a spikelet/grain counting tool, and a tool for the analysis of seed shape. The three tools can be used independently or simultaneously for analysis of the same image. Results are then reported in the Extensible Markup Language (XML) and Comma Separated Values (CSV) file formats. Images of rice panicles were used to evaluate the efficiency and robustness of the software. Compared to data obtained by manual processing, P-TRAP produced reliable results in a much shorter time. In addition, manual processing is not repeatable because dry panicles are vulnerable to damage. The software is very useful, practical and collects much more data than human operators. It has been used in many centers around the world and despite being old (developed in 2013) and with little maintenance and upgrade, it still one of the top and most used systems in panicle image analysis. There are more powerful and precise new systems that use the state-of-art image processing libs and more sophisticated algorithms but P-Trap is platform-independent, user-centered, and easy to use with fantastic GUI. See it here: https://www.youtube.com/watch?v=o5K1pffy1v8

  • Iterative illumination correction with implicit regularization:
    ill-posed inverse problems, image degradation, iterative least-squares, boosting, QR-factorization, R-squared
    article|source code

    A retrospective algorithm for correcting the uneven illumination field in microscopy images. The illumination field is iteratively made uniform using an increasing sequence of bivariate polynomials. At each iteration, the least squares problem of fitting a 2-D polynomial to a sampled image is solved by using QR decomposition with column pivoting, where image samples are obtained by dynamic programming or watershed transform. This incremental scheme allows the smoothness constraint of the estimated bias field to be implicitly satisfied. The proper number of iterations is determined by an automatic stopping criterion. This algorithm is similar to the Gradient Boosting with stopping criterion where the R-squared is used to determine the deterioration in the correction boosting.
